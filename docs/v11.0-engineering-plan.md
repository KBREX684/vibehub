## 〇. 怎么读这份计划

### 〇.1 三个铁律（保证两个 AI 不打架）

1. **契约先行**：每个 P-阶段开工前，先冻结一份"接口契约表"——OpenAPI 路径、TS 类型、Fixture JSON。冻结后 BE 与 FE 各自闭门干活，**不互相催**。
2. **Fixture 驱动**：FE 不等 BE。所有 API 都预先在 `web/src/lib/data/mock-*` 写一份 fixture；FE 用 `USE_MOCK_DATA=true` 跑通即视为完成，**不要求联调**。
3. **Sync Gate**：每个 P-阶段末有一个集成 Gate，由真实接口替换 fixture 跑 smoke。Gate 不通过则阶段不结束，**不允许进入下一阶段**。

### 〇.2 AI 分工（不交叉）

| 角色 | 工具 | 负责范围 |
| --- | --- | --- |
| **BE** | GPT | `web/prisma/**`、`web/src/app/api/v1/**`、`web/src/lib/repositories/**`、`web/src/lib/repository.ts`、`web/src/lib/openapi-spec*.ts`、`web/tests/**`（API/repo 单测） |
| **FE** | GLM | `web/src/app/**/page.tsx`、`web/src/components/**`、`web/src/locales/**`、`web/src/lib/data/mock-*`（fixture）、`web/tests/e2e/**`（UI E2E） |
| **共担** | 你（PM） | 契约文件、Pricing 文案、文档归档、PR review、合并冲突仲裁 |

**红线**：
- **GPT 不动 `web/src/components/**` 和 `web/src/app/**/page.tsx`**
- **GLM 不动 `web/prisma/**` 和 `web/src/app/api/**`**
- **共享文件**（`web/src/lib/types.ts`、`web/src/lib/generated/api-types.ts`、`web/src/locales/zh.json`）由你这边手动 merge，禁止两个 AI 直接改

### 〇.3 任务粒度

每条任务都是"**一个 prompt 让 AI 一次做完**"的粒度。每条任务 = 一个独立 commit。

### 〇.4 任务命名约定

```
[阶段ID-序号][BE|FE|CONTRACT] 简短动词标题
   例：[P1-03][BE] LedgerEntry Prisma model + migration
   例：[P1-04][FE] /ledger page skeleton with mock fixture
   例：[P1-01][CONTRACT] Freeze ledger API surface
```

### 〇.5 每条任务的标准 prompt 模板

> 你可以把任务卡 1:1 复制粘贴到 GPT / GLM 里。

```text

请只完成下面这一项任务，不要顺手改其他文件。

  - 不修改 [对方负责区域]
  - 共享文件 [...] 不能动，需要我来 merge

  - 接口路径：...

```

### 〇.6 阶段时间盒（建议）

每个 P-阶段 1–2 周交付。如果某条任务 AI 一次做不完，**拆成两条**而不是延期。

```ts
// LedgerEntry TS 类型（同 prisma model）
export interface LedgerEntry {
  id: string;
  workspaceId: string;
  actorType: 'user' | 'agent';
  actorId: string;
  actionKind: string;        // e.g. 'workspace.artifact.upload'
  targetType: string | null;
  targetId: string | null;
  payloadHash: string;       // 64 hex
  prevHash: string | null;   // 64 hex 或 null
  signature: string;         // base64 ed25519 sig
  signedAt: string;          // ISO
  anchorChain: 'vibehub' | 'zhixin' | 'baoquan';
  anchorTxId: string | null;
  anchorVerifiedAt: string | null;
}

// 列表响应
export interface LedgerListResponse {
  items: LedgerEntry[];
  cursor: string | null;
}

// 校验响应
export interface LedgerVerifyResponse {
  ok: boolean;
  brokenAt: string | null;       // 第一个 hash 校验失败的 entry id
  totalChecked: number;
  signedBy: 'vibehub-signer-v1';
}

// API 列表
GET    /api/v1/me/ledger?workspaceId=&from=&to=&cursor=&limit=
GET    /api/v1/workspaces/:id/ledger
GET    /api/v1/workspaces/:id/ledger/export?format=json
GET    /api/v1/ledger/:id/verify              # 公开
POST   /api/v1/ledger/verify-bundle           # 公开，body=ledger.json

// Fixture 文件
web/src/lib/data/mock-ledger.ts
  → 至少 50 条 entry，覆盖 actionKind:
    workspace.artifact.upload
    snapshot.create
    deliverable.approve
    agent.task.complete
    aigc.stamp.apply
```
