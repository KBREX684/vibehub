## 0. PM 必读：为什么不一次性全站切换

仓库当前的 dark Monochrome Geek 是 v8/v9/v10 三个版本累计沉淀的设计语言，包括：
- `globals.css` 的 60+ 个暗色 token
- 50+ 个组件按 border-first elevation 设计（在白底上视觉密度会过高）
- 部分 v8 组件有硬编码 `bg-[#xxx]`，绕过了 token

如果一次性全站切，会出现：
- 暗色卡片在亮底上"飘"
- 印戳虚线方框失去对比
- 老 admin 页突然"瞎"

所以本次只动 v11.0 已落地的 17 个文件（页面 6 + 组件 11），其他保持暗色。两套主题通过 `data-theme` 切换，互不干扰。

## 1. 设计 DNA（v11.1 暖色版，必须冻结）

### 1.1 一句话哲学

> **VibeHub v11.1 = "纸感工作台"。** 让独立开发者长时间工作时眼睛不累，让 Ledger 像翻一份纸质账本，让操作像盖章一样安心。

参照 DNA（按权重）：
- **Claude.ai / Claude Code 主界面 50%** — 米白底 + 暖灰文字 + Anthropic 砖橙
- **Linear marketing 浅色 20%** — 排版节奏 + 阴影克制
- **Stripe Docs / Vercel docs 浅色 15%** — 表格、代码块、卡片
- **Notion 默认页面 10%** — 长阅读舒适度
- **Apple Human Interface Light 5%** — 圆角与系统字体堆栈

**禁止**：
- 不做高饱和（任何 #FF / #00 起头的纯色）
- 不做 glassmorphism / backdrop-blur > 6px
- 不做 neumorphism（凸起 / 凹陷）
- 不做漫画风插画
- 不做政务/公章式视觉
- 不做 Material Design 的色块卡片（FAB、Ripple 等）

### 1.2 暖色护眼基准（这一节就是给设计师定底色）

按 Claude.ai 的实际取色（暖灰偏米色，色温 ~5400K）：

| 用途 | 颜色 | 描述 |
| --- | --- | --- |
| 全局画布 | `#FAF9F6` | 米白（warm off-white），不是纯白 #FFF |
| 主面 / 卡片 | `#F5F2EC` | 比画布稍深的暖米 |
| 抬升面 / Modal | `#FFFFFF` | 纯白只用在抬升层（这是 Claude 做法） |
| 次要画布 / 段落分隔 | `#EFEBE3` | |
| Hover surface | `#EBE6DC` | |
| Subtle 背景 | `rgba(60, 40, 20, 0.04)` | 一抹暖棕透明 |
| 主要文字 | `#1F1B16` | 深暖棕（不是 #000） |
| 次要文字 | `#5C544A` | 暖灰棕 |
| 三级文字 | `#8B8378` | |
| 辅助/占位 | `#B5AC9E` | |
| 反白文字（在主色按钮上） | `#FAF9F6` | 用画布色，不用纯白 |
| 默认边框 | `rgba(60, 40, 20, 0.10)` | |
| 强边框 | `rgba(60, 40, 20, 0.18)` | |
| Focus 环 | `rgba(196, 90, 47, 0.45)` | 用主色 50% alpha |

**主色（CTA、链接、focus）**：
- `--color-accent-primary: #C45A2F`（Claude 砖橙 / Anthropic Sienna）
- `--color-accent-primary-hover: #A94A24`
- `--color-accent-primary-subtle: rgba(196, 90, 47, 0.10)`
- `--color-accent-primary-border: rgba(196, 90, 47, 0.30)`

**辅助色（语义保持，但全部往暖色调一档）**：
- `--color-success: #5B7A3F`（暖橄榄绿，不是 #34d399）
- `--color-warning: #C68A2E`（暖琥珀）
- `--color-error: #B23A3A`（暖砖红）
- `--color-info: #6A7AA0`（暖石青）
- 每个色都配 `-subtle`（10% alpha）和 `-border`（25% alpha）

**v11 印戳元素的暖色重映射**：
- 默认（未锚定）：1px dashed `--color-text-tertiary`，文字 `--color-text-tertiary`
- 已锚定到司法链：1px dashed `--color-success`，背景 `--color-success-subtle`
- 校验失败：1px dashed `--color-error`，背景 `--color-error-subtle`

### 1.3 字体（保持 Geist Mono，但 sans-serif 换软一点的字体）

```
--font-sans: "Inter", "PingFang SC", "Source Han Sans SC", -apple-system,
             BlinkMacSystemFont, "Segoe UI", system-ui, sans-serif;
--font-mono: "Geist Mono", "JetBrains Mono", "Source Code Pro",
             ui-monospace, SFMono-Regular, Menlo, monospace;
--font-serif: "Source Han Serif SC", "Noto Serif SC", Georgia, serif;
```

**v11.1 新增**：`--font-serif` 仅用于 Ledger PDF 渲染与 Trust Card 标题（让"账本"质感更纸感）。其他全部 sans。

字体大小不变，但**全局行高 +0.05**：
- 默认行高 1.65（v11.0 是 1.6）
- 段落 line-height 1.75（提升长阅读舒适度）

### 1.4 半径与阴影（这是与暗色版最大的差异）

暗色 Monochrome Geek 是 border-first，没有阴影。**暖色护眼必须用 soft shadow elevation**，否则白底上的卡片会"飘"。

```
--radius-xs:    4px       (v11.0 是 2px，全部 +2 增加柔和感)
--radius-sm:    6px
--radius-md:    10px
--radius-lg:    14px
--radius-xl:    18px
--radius-2xl:   24px
--radius-3xl:   32px
--radius-pill:  9999px

--shadow-card:        0 1px 2px rgba(60, 40, 20, 0.04);
--shadow-card-hover:  0 2px 6px rgba(60, 40, 20, 0.08);
--shadow-elevated:    0 8px 24px rgba(60, 40, 20, 0.10);
--shadow-modal:       0 24px 64px rgba(60, 40, 20, 0.18);
--shadow-focus-ring:  0 0 0 3px var(--color-focus-ring);
```

**关键**：阴影色不是黑色（`rgba(0,0,0,...)`），是暖棕色（`rgba(60, 40, 20, ...)`）。这一条决定了整体气质能否做到 Claude 那种"安静"。

### 1.5 间距与节奏（更宽松）

8pt grid 不变，但 v11.1 默认间距全部 +1 档：

| 用途 | v11.0 暗色 | v11.1 暖色 |
| --- | --- | --- |
| Card padding | 16-20 | 20-24 |
| Section gap | 32-64 | 48-96 |
| Form field 行距 | 12 | 16 |
| 段落间距 | 16 | 20 |

理由：暖色背景对比度低，需要更多负空间避免拥挤。这是 Claude / Notion 的关键差异。

## 2. 动效系统（"丝滑平滑"= 缓动函数 + 持续时间，不是动效堆叠）

设计师常误解"丝滑"= 加更多动画。**真正的丝滑 = 几乎察觉不到动效**，但每一次 hover / focus / 出现都有微小变化。

### 2.1 全站缓动函数（统一）

```css
/* 在 globals.css 顶部定义 */
--ease-standard:   cubic-bezier(0.32, 0.72, 0, 1);   /* 进入/离开默认 */
--ease-accelerate: cubic-bezier(0.4, 0, 1, 1);       /* 离开 */
--ease-decelerate: cubic-bezier(0, 0, 0.2, 1);       /* 进入 */
--ease-spring:     cubic-bezier(0.34, 1.56, 0.64, 1); /* 仅按钮弹跳 */
```

**禁止**：`linear`、`ease-in-out` 标准 CSS 缓动。它们看起来都很"机械"。

### 2.2 持续时间表（统一，不允许临时拍脑袋）

| 行为 | 时长 | 缓动 |
| --- | --- | --- |
| Hover 颜色变化 | 120ms | `--ease-standard` |
| Hover 阴影变化 | 200ms | `--ease-decelerate` |
| Card 出现 / 上滑 | 320ms | `--ease-decelerate` |
| Modal 打开 | 240ms | `--ease-decelerate` |
| Modal 关闭 | 180ms | `--ease-accelerate` |
| Drawer 打开 | 320ms | `--ease-standard` |
| Tab 指示器移动 | 280ms | `--ease-standard` |
| Toast 进入 / 退出 | 240 / 180ms | `--ease-decelerate` / `--ease-accelerate` |
| 按钮按下回弹 | 280ms | `--ease-spring` |
| 列表项渐入（≤ 8 项） | 200ms + 30ms stagger | `--ease-decelerate` |

### 2.3 微动效规则（这是丝滑的关键）

- **任何颜色变化都必须有 transition**，不要让颜色"瞬切"
- **任何 hover 都必须同时有 1-2px 的 transform**（如 `translateY(-1px)`）
- **任何按钮都必须有 active 态的 `transform: scale(0.98)`**，给"按下去"的物理感
- **scroll 时**，stick 元素的阴影应该按 scrollY 渐入（前 20px 内 0→1）
- **Skeleton 加载**用 1.4s 缓慢循环，不要 < 1s（会闪眼）

### 2.4 必须禁止的动效

- 视差滚动（parallax）
- conic / radial gradient 动画
- 装饰性粒子背景
- 任何"光晕"" glow"
- 持续旋转的 loading spinner（用 progress dots 或 skeleton 代替）
- 弹性大于 `--ease-spring` 的弹跳
- 持续时间 > 400ms 的进入动效（拖累感知速度）

### 2.5 prefers-reduced-motion

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

这一段必须放在 `globals.css` 末尾。无障碍硬性要求。

## 3. 布局 / 留白原则（清爽、舒适、干净）

### 3.1 容器宽度（v11.1 收紧）

| 容器类 | v11.0 | v11.1 |
| --- | --- | --- |
| `.container` | `min(1280px, 94vw)` | `min(1120px, 92vw)` |
| `.container-narrow`（新增） | — | `min(720px, 90vw)`（用于阅读型页：Home/Pricing/Settings） |
| `.container-wide`（新增） | — | `min(1440px, 96vw)`（仅 Studio / Ledger 主区） |

### 3.2 标题 / 正文节奏

- H1 与正文之间至少 **24px** 间距
- H2 与上方区块至少 **64px**（v11.0 是 48）
- H3 与正文 **16px**
- 段落之间 **20px**
- 表格行高 **44px**（v11.0 是 36）

### 3.3 单页焦点原则（最重要）

**每个页面只允许有 1 个主要 CTA 颜色**（`--color-accent-primary`）。
其他动作一律 ghost / text-only。

→ 例如 Pricing 页：Free 卡的 CTA 是 ghost，只有 Pro 卡的 "升级 Pro" 用主色。
→ Studio 页：右上角"新建" 用主色，其他全 ghost。
→ Ledger 页：导出按钮 ghost，没有主色。

这一条是清爽感的来源——视觉热点必须稀缺。

### 3.4 卡片密度

- 列表页每屏可见 **5-7 张卡片**（不是 v11.0 的 8-10 张）
- 卡片之间间距 **20px**
- 卡片内部 padding **20-24px**
- 卡片之间不允许 divider（间距已经够区分）

### 3.5 表格规则

- 表头：`--color-bg-canvas` 米白，**不是** subtle 阴影
- 行：默认无背景；hover 时 `--color-bg-subtle`
- 行高 44px
- 表头字体：Geist Mono，11px，uppercase，`--color-text-tertiary`
- 单元格分隔：仅底部 1px `--color-border-subtle`，不要竖线
- 数字列右对齐 + Geist Mono

```css
[data-theme="warm"] {
  --color-bg-canvas: #FAF9F6;
  --color-bg-surface: #F5F2EC;
  --color-bg-elevated: #FFFFFF;
  /* ... 全部 warm token ... */
  color-scheme: light;
}

/* 老 admin / work 路由保留 dark */
[data-theme="dark"] {
  --color-bg-canvas: #09090B;
  --color-bg-surface: #101014;
  --color-bg-elevated: #16161C;
  /* ... 全部 dark token，复制自 v11.0 globals.css ... */
  color-scheme: dark;
}
```

### 7.2 `web/src/app/layout.tsx`

```tsx
export default function RootLayout({ children }) {
  return (
    <html lang="zh" data-theme="warm">  {/* 默认 warm */}
      <body>{children}</body>
    </html>
  );
}
```

### 7.3 老路由局部覆盖

`web/src/app/admin/layout.tsx` 和 `web/src/app/work/layout.tsx` 各自加：

```tsx
<div data-theme="dark" className="min-h-screen">
  {children}
</div>
```

这样 v11 主线全暖色，老路由仍保留暗色，互不干扰。

## 8. 必须的工程红线

1. **不动 `web/src/app/admin/**`、`web/src/app/work/**`（除 `/work/page.tsx`）、`web/src/app/checkout/**`**
2. **任何颜色都通过 CSS 变量引用**（`var(--color-*)`），不允许在组件里写 hex
3. **任何阴影都用 `--shadow-*` token**，不允许临时写 `shadow-[0_2px_4px_...]`
4. **任何动效都从 §2 的 5 条缓动 + 持续时间表里选**，不允许临时拍脑袋
5. **每页只允许 1 个主色 CTA**（§3.3）
6. **每个 PR 一个组件 / 一个页面**，不要批量提交
7. **提交前必须本地 `npm run lint && npx tsc --noEmit && npm run build` 通过**
8. **暖色版与暗色版必须能共存**（双主题），不允许通过删除 dark token 完成切换
