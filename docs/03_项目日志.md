## 2026-04-12 P2-5 Log (weekly leaderboard materialization)

- Goal: ship **time-windowed weekly rankings** with optional **materialized snapshots** so public pages stay stable after the week closes.
- Actions: added Prisma snapshot tables; `getWeeklyDiscussionLeaderboard` / `getWeeklyProjectCollaborationLeaderboard`; `getWeeklyLeaderboardPublicPayload` (prefers materialized rows); admin materialize endpoint + audit log; public weekly leaderboard APIs; extended `/leaderboards` with week navigation; admin dashboard materialize form; Vitest coverage for UTC week parsing and mock materialization path.
- Outputs: migration under `web/prisma/migrations/`, new API routes, repository helpers, UI updates, `tests/p2-5-weekly-leaderboards.test.ts`.
- Risk: week boundary is **UTC Monday** — product copy documents this; cross-timezone “local week” is out of scope for this slice.
- Decision: when no snapshot exists, public APIs fall back to **live** weekly counts so empty databases still behave sensibly.
- Next: staff picks / featured projects, challenge campaign pages, or MCP parity for discover filters.
