## 第一部分：公司介绍（对外一页纸）

### 我们是谁

**VibeHub** 是一家面向中文开发者生态的 **开发者社区与交付协作平台** 创业公司。我们以「独立开发者与小团队」为第一用户，把 **讨论沉淀、作品展示、组队协作、可审计的 Agent 参与** 串成一条可复用的主路径，而不是只做单一形态的内容站或工具站。

### 我们在解决什么问题

- **信息碎片化**：技术讨论与项目成果分散在多个渠道，难以形成可检索、可跟进的「作品与协作」资产。
- **协作起点高**：陌生人之间的可信组队、权限边界与交付节奏，缺少与社区身份绑定的低摩擦入口。
- **人机协作缺治理**：Agent 能力普及后，团队更需要 **合法、受控、可审计** 的机器参与方式，而不是无限权限的「黑箱调用」。

### 我们的产品是什么

VibeHub 当前产品主线围绕四层闭环建设：

1. **公开发现**：讨论区、项目画廊、创作者主页，形成高质量供给与互动。
2. **协作入口**：结构化的合作意向与申请通道，降低陌生人协作的启动成本。
3. **协作中枢（战略核心）**：**Team Workspace** — 小团队（2–10 人优先）在云端共享上下文、里程碑与交付资产，本地开发习惯不变。
4. **受控执行**：Agent 以 **权限矩阵、限制策略、确认流与审计** 的方式进入团队工作流；合规与治理能力是产品能力的一部分，而非事后补丁。

商业化采用清晰的 **Free + Pro** 个人订阅；企业相关能力定位为 **认证与治理辅助的次要层**，不抢占核心飞轮。

### 我们的差异化

- **开发者优先、小团队优先**：不做 IDE、不把代码托管作为主路径，与垂直工具错位竞争。
- **结构化资产 + 开放接口**：REST API、OpenAPI、MCP 等开发者接口，使人类创作与机器可消费数据同源。
- **中国语境下的合规与可见化**：将内容治理、数据区域与风险可见性纳入产品叙事与路线图（V9.0）。

### 我们追求的终局指标（North Star）

在中文 VibeCoding 与独立开发者生态中，成为 **「人类协作网络 + Agent 可调用知识基础设施」** 的高密度入口：讨论与项目可被持续发现，协作可被追踪，机器参与可被治理。

## 第二部分：商业计划书（BP）

### 1. 执行摘要

VibeHub 以 **社区供给 → 项目展示 → 团队协作 → 合规交付** 为主增长飞轮，通过 **订阅（Free | Pro）** 与未来的 **增值能力（如更高阶的 Workspace 能力、接口配额等）** 实现收入。技术栈以 **Next.js 全栈应用 + PostgreSQL（Prisma）** 为核心，强调 **真实数据路径上的质量门禁**（迁移、种子数据、冒烟与 CI 门禁），确保产品迭代可持续。

当前战略阶段 **V9.0** 的重点不是简单堆功能，而是：将 **Team** 升格为 **Team Workspace**，升级合作意向为 **合规协作通道**，将 Agent 从「可调用」推进到 **「可控协作」**，并把治理与中国化能力转化 **用户可感知** 的平台体验。

### 2. 市场机会

**需求侧（2026）**

- AI 编程与 Agent 工作流快速渗透，开发者对 **可信实践、结构化项目资产、可协作网络** 的需求同步上升。
- 中文开发者生态仍缺乏 **「讨论—展示—协作—机器可消费」一体化** 的默认平台。

**供给侧缺口**

- 泛社区偏流量与碎片交流，难以沉淀可检索的项目与协作记录。
- 作品展示站偏静态，缺少持续协作与交付闭环。
- 传统协作工具偏「已组队团队内部」，对 **陌生人组队与信誉机制** 支持不足。
- Agent 侧缺稳定 schema 与权限治理，导致上下文质量与风险不可控。

**目标细分市场**

- 首选：**中文开发者、独立开发者、2–10 人小团队 / 工作室**。
- 次级：**需要发现与观察项目信号的企业或资本侧用户**（保持轻量、非主产品线）。

### 3. 产品与服务

**核心模块（与仓库现状一致）**

| 模块 | 价值 |
|------|------|
| 社区与讨论 | 沉淀方法、复盘与互动，形成复访与话题供给 |
| 项目画廊与创作者主页 | 可发现、可对比、可持续更新的作品与履历 |
| 团队协作 | 团队、加入申请、里程碑、任务、活动记录等 |
| 开发者平台 | API Keys、OpenAPI、MCP、公共 API 镜像，服务工具链与 Agent |
| 商业化 | Free / Pro、计费与权益闸门；Pro 聚焦「空间与曝光」等可量化价值 |
| 治理与合规 | 内容审核、审计日志、企业认证徽章、风控与合规页面（持续迭代） |

**路线图要点（V9.0）**

- Workspace 数据层与页面承接；**Snapshot Capsule** 等资产快照能力。
- **CollaborationIntent v2** 与「三句式」合作申请体验。
- **Agent × Workspace** 读写矩阵、统一活动流与确认流。
- **Restriction** 三原语与合规可见化，落实「可审计协作」。

**明确冻结（执行期减少扩散）**

- OAuth Apps、Automation、Webhook、外部连接器等的 **扩张型新主线** 默认冻结，以保证 V9.0 交付带宽（详见 `docs/roadmap-current.md`）。

### 4. 商业模式与单位经济（框架）

**收入来源**

1. **Pro 个人订阅**：更多项目空间、曝光与开发者工具增强（具体权益以产品配置为准）。
2. **未来增值**：Team Workspace 相关高阶能力、API / Agent 调用配额等（在 PMF 与治理就绪后逐步释放）。

**单位经济示意公式**

- \( \text{MRR} \approx \text{MAU} \times \text{Pro 转化率} \times \text{ARPU} \)

**关键运营指标**

- Pro 转化率、Pro 留存、活跃创作者数、项目页完整度与更新率、协作意向成功率、Workspace 活跃团队数、接口与 Agent 调用的健康度与滥用率。

定价原则强调 **仅保留 Free 与 Pro 两档** 时的低摩擦决策；Free 不锁基础社区参与能力，Pro 售卖「增量价值」而非「基础解锁」。（详见既有策略文档 `VibeHub_项目计划书_v3.0.md` / v4.0 思路。）

### 5. 竞争与定位

**不正面竞争**

- 通用社交平台的总流量。
- 单一 AI IDE 的编辑器能力。
- 以代码托管与 CI 为主轴的 DevOps 平台。

**竞争策略**

- 聚焦 **高质量开发协作密度** 与 **结构化资产**。
- 以 **开放接口 + 治理** 建立复利：项目与人才可被机器安全消费。

### 6. 技术、数据与安全

- **栈**：Next.js、TypeScript、PostgreSQL、Prisma；配套 Redis、任务队列、对象存储等可按环境扩展。
- **质量门禁**：Lint、测试、OpenAPI 校验、类型生成与构建；CI 侧对齐 **真实数据库** 的迁移、种子与冒烟路径。
- **安全与治理**：会话与密钥管理、API Key 分域与速率限制、Webhook 与上传等敏感路径的安全测试覆盖；运营侧审计与后台治理能力持续投入。

### 7. 合规与风险管理（摘要）

- 生成式 AI、算法推荐、ICP/备案、个人信息与数据安全等 **按业务场景分阶段对齐**。
- 平台侧：内容分级治理、申诉与审计；Agent 侧：白名单、最小权限、关键操作确认。
- 协作场景：纠纷与资产归属的规则模板与责任边界声明。

### 8. 里程碑与融资用途（模板）

**近阶段里程碑（与 V9.0 分段一致，可改为具体日期）**

- P0：边界校正、设计系统收口与页面骨架统一。
- P1：Workspace 数据层、页面与订阅承接。
- P2：合作申请迁移、Snapshot Capsule MVP。
- P3：Agent × Workspace 与统一活动流 / 确认流。
- P4：Restriction 与合规可见化收口。

**资金用途（示例）**

- 产品与工程：Workspace、治理与体验闭环。
- 社区与运营：种子创作者与团队计划、主题活动与榜单。
- 合规与安全：法务、审计与基础设施。
- 商业化：支付、风控与增长实验。
