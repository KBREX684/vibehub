# VibeHub Roadmap History

This file separates historical planning material from the current execution line.

## Historical sources retained

- `docs/01_实现计划图.md` — detailed implementation chronology and stage-by-stage history
- `docs/03_项目日志.md` — long-form execution log
- `VibeHub_项目计划书_v3.0.md` — strategic planning document (content is v4.0; filename retained for link stability)
- `docs/roadmap-v5.md` — v5.0 full-stack evolution roadmap (superseded by v7.0)
- `docs/08_backend_optimization_roadmap.md` — backend optimization roadmap v1.0

## Why these files are still kept

- They document migration decisions, phased delivery history, and prior acceptance context.
- Several audit notes still point at them as historical evidence.
- They remain useful when reviewing old migrations, test additions, and strategic pivots.

## Current policy

- Do not use the files above as the primary source of truth for current strategy.
- Use `docs/roadmap-current.md` for current direction.
- Use `docs/roadmap-v7.md` for the active production upgrade plan (v7.0).
- Use `docs/release-notes.md` for recent product-facing changes.
- Use `docs/repository-cleanup-report.md` for repository closure decisions.
