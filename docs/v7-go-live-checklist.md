# V7.0 Go-Live Checklist

更新日期：2026-04-17  
适用范围：`/root/vibehub/web` 当前 `V7.0` 收尾后的正式上线准备  
判定原则：这份清单是 `docs/launch-readiness-standard.md` 的执行版，不替代标准，只负责把上线动作落到可勾选项。

---

## 1. 目标

在正式上线前，用一份可执行、可复核、可追责的清单确认：

1. 代码、数据库、配置、外部依赖和运维手册都已经到位。
2. 核心用户链路、核心管理员链路和开发者接入链路都可真实工作。
3. 不把“代码已写完”误判成“可以正式上线”。

---

## 2. 现状

当前仓库状态：

- `V7.0` 主体功能已完成收尾。
- 单元/集成测试已通过。
- 构建已通过。
- fresh-schema `migrate deploy + seed + smoke:live-data` 已通过。
- 全量 Playwright 浏览器模拟测试已通过。

当前仍需人工确认的不是代码缺口，而是上线前置条件、外部配置和发布动作。

---

## 3. 风险

正式上线前的主要风险已经从“代码回归”转为“环境与运营失配”：

1. 生产环境变量或 secrets 配置不完整，导致 fail-fast 或带病启动。
2. 中国支付、SMTP、Webhook、OAuth 回调域名等外部配置未切到正式值。
3. 合规项已识别但未完成最终确认。
4. 数据迁移虽然可执行，但没有按正式顺序演练，发布窗口仍可能出错。
5. 回滚和人工值守安排不清晰，出现问题时没人接。

---

## 4. 实施步骤

以下清单默认按上线前顺序执行。

### A. 发布窗口准备

- [ ] 明确本次上线负责人、技术值守人、业务确认人

- [ ] 冻结非上线必要改动
- [ ] 确认当前待发布 commit / tag
- [ ] 确认数据库备份策略和回滚负责人

### B. 代码与构建基线

- [ ] 在待发布代码上执行 `npm run lint`
- [ ] 在待发布代码上执行 `npm run validate:openapi`
- [ ] 在待发布代码上执行 `npm run generate:types`
- [ ] 在待发布代码上执行 `npm test`
- [ ] 在待发布代码上执行 `npm run build`

## 5. 验收标准

只有以下条件同时满足，才允许判定 `V7.0` 可正式上线：

1. `launch-readiness-standard` 中的硬门槛没有未通过项。
2. 本清单中 `A-K` 所有必选项均已勾选。
3. `L` 观察项已经有明确的负责人和观察窗口。
4. 未关闭的问题只允许是低风险工程债，且不影响：
   - 登录
   - 讨论
   - 项目
   - 团队
   - API/MCP
   - 支付/订阅
   - 后台治理

如果以上任意一项不满足，则结论只能是：

**暂不正式上线，继续补齐阻塞项。**
