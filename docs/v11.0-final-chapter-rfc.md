## 一. 走过的路（客观复盘）

### 1.1 v8（2026-Q1）：五段并列的诱惑

定位："中国中文开发者 AI+Human 协作网络"
顶栏：广场讨论 / 项目 / 团队 / 开发者 / 定价
问题：五段同时打 5 个红海（即刻 / DecoHack / 飞书 / GitHub Explore / Notion），全输。

### 1.2 v9（2026-Q2）：Team Workspace 协作中枢

定位："本地开发之外的人与 Agent 协作中枢"
新增数据：`WorkspaceArtifact` / `SnapshotCapsule` / `Restriction`
问题：Team Workspace 撞飞书 / Linear，一个独立开发者团队拼不过资源压倒性的巨头。

### 1.3 v10（2026-04-18 落地于 main）：Workspace-first 主线

落地：`/work/**` 9 路由、`Workspace / SnapshotCapsule / WorkspaceArtifact / WorkspaceDeliverable / AgentTask` schema、
`workspace.repository.ts` 1778 行、ConsoleShell + 三大面板。
价值：把 Agent 治理从"可调用"升级为"可控协作"。
问题：仍然在做 Team Workspace + Discover + Project Library + Intent，**主线不够窄**。

### 1.4 v10.5 候选（公证人方向）

提议把 v10 的 Workspace + AgentActionAudit + AgentConfirmationRequest + SnapshotCapsule + Deliverable
**重新解读**为"AI 协作公证人"，新增三创新（合规 / 账本 / 信用）。
问题：三创新一起做仍然是"3 件事并列"，不符合"成熟公司只做一件"的纪律。

### 1.5 候选路四维评分

按"专精 / 护城河 / 落地简单 / 推广容易"（每项 0-5）：

| 候选路 | 专精 | 护城河 | 落地 | 推广 | 总分 |
| --- | --- | --- | --- | --- | --- |
| Personal Workspace + Agent 协作（v10 主线） | 3 | 2 | 5 | 2 | 12 |
| Team Workspace 团队协作 | 3 | 1 | 3 | 2 | 9 |
| OPC × OPC 撮合 | 3 | 2 | 3 | 4 | 12 |
| 项目展示 / Discover 主路径 | 3 | 1 | 4 | 3 | 11 |
| AIGC 合规中间件 | 4 | 2 | 5 | 5 | 16 |
| **AI 操作公证账本（创新一）** | **5** | **5** | **3** | **3** | **16** |
| OPC 信用名片 | 4 | 4 | 3 | 3 | 14 |

合规中间件与操作公证账本并列 16 分。决定胜负的是**持续性**与**网络效应依赖**：

- 合规中间件：腾讯云 / 阿里云已有 API（0.05 元/条），24 个月内大厂下场，护城河蒸发。
- 操作公证账本：腾讯阿里**永远不会做**（太薄、单价低、不在云资源审计层），且**单用户独立有价值**，0 网络效应依赖。

→ **v11.0 唯一主轴 = AI 操作公证账本（Ledger）。合规中间件与信用名片作为它的"前置插件"和"后置出口"，不单独叙事。**

## 二. 一句话定位（冻结）

中文：
> **VibeHub — 你和 AI 一起做的工作，有据可查。**

英文：
> **VibeHub — Your AI work, on the record.**

副标（解释一句话）：
> 在 VibeHub 工作站里和 Agent 一起做的每一件事，自动加合规标识、自动留可校验账本、自动沉淀为可外发的信用名片。

## 三. 用户与场景

### 3.1 唯一目标人群

- **中国独立开发者 / OPC（一人公司）创始人**
- 75% 非技术背景，使用 Cursor / Claude Code / Codex 等 AI 工具完成日常工作
- 体量：腾讯云《2026 OPC 报告》数据 → 全国 1600+ 万家一人有限责任公司，2025H1 新增 286 万

### 3.2 高频场景（每日）

在 `/studio` 里和 Agent 干活：
- 提交任务 → Agent 执行 → 人工确认 → 输出
- 每个动作自动写入 Ledger
- 每个产出物自动加 AIGC 标识

### 3.3 中频场景（每周）

导出周度 Ledger 给客户 / 合作方：
- `/ledger` 选择时间范围 → 导出 `ledger.pdf`
- PDF 上含 hash 校验码 + 可选已锚定司法链信息

### 3.4 低频场景（每月）

- 更新 Trust Card 公开内容
- 申请政府 OPC 扶持时附 Ledger 月度合规报告
- 接外包时把 Trust Card 链接发给客户做背书
- 投资人尽调时一键拉协作历史

## 四. 三件事架构

### 4.1 Studio（做）— `/studio`

个人 AI 工作站（v10 Personal Workspace 升级）。

- 任务流：任务 + Agent 执行卡片
- 文件 / artifact
- 快照
- 绑定 Agent 列表（MCP / API Key）
- **底部固定 metric chip**：本月 Ledger 条目数 + AIGC 标识覆盖率

### 4.2 Ledger（签）— `/ledger`

操作账本（**v11 核心新增**）。

- 全部 Agent 写入 + 人工确认 + 快照创建 + 交付通过事件的时间线
- 每条带 hash 短码 + "已锚定"徽章
- 一键导出 `ledger.json` / `.pdf` / `.txt`
- 提供 `vibehub-verify` CLI 独立校验
- 月度合规报告导出（GB 45438-2025 格式）

### 4.3 Card（晒）— `/u/[slug]`

公开信用名片（OPC Trust Card）。

- 头像 + 一句话定位 + 公司类型徽章
- **6 个真实指标**（全部来自 Ledger，禁止手填）：
  - 累计 Ledger 条目数
  - 累计快照数
  - 已加 AIGC 标识 artifact 数
  - 公开作品数
  - 平均响应时长
  - 注册时长
- 公开作品集（Studio 公开部分）
- "下载我的 Trust Card PDF"按钮（PDF 含 hash 校验码）

### 4.4 三者因果

```
        ┌─────────────────────────────┐
        │  Studio（做）                │ ← 入口（高频，每日）
        └────────────┬────────────────┘
                     │ 操作行为
                     ▼
        ┌─────────────────────────────┐
        │  Ledger（签）                │ ← 沉淀（中频，每周）
        └────────────┬────────────────┘
                     │ 凭证累积
                     ▼
        ┌─────────────────────────────┐
        │  Card（晒）                  │ ← 出口（低频，每月）
        └─────────────────────────────┘
```

### 4.5 不在主线（明确除外）

- 团队多人协作 / 私信 / 群聊
- 项目发现 / 撮合 / 配对
- 论坛 / 帖子 / 排行榜 / 内容流
- 自建 LLM / Agent 商店 / Prompt 市场
- IDE / repo / CI

## 六. 数据层（最终）

### 6.1 新增 5 张表

```prisma
model LedgerEntry {
  id              String   @id @default(cuid())
  workspaceId     String                        // 仅 personal 类型
  actorType       LedgerActorType               // user | agent
  actorId         String                        // userId 或 agentBindingId
  actionKind      String                        // workspace.artifact.upload | snapshot.create | deliverable.approve | agent.task.complete | aigc.stamp.apply 等
  targetType      String?                       // artifact | snapshot | deliverable | agentTask | task
  targetId        String?
  payloadHash     String                        // SHA-256(payload JSON)
  prevHash        String?                       // 上一条的 signature；首条为 null
  signature       String                        // ed25519 签名 = sign(payloadHash || prevHash)
  signedAt        DateTime @default(now())
  anchorChain     LedgerAnchorChain @default(vibehub)
  anchorTxId      String?
  anchorVerifiedAt DateTime?

  @@index([workspaceId, signedAt])
  @@index([actorType, actorId, signedAt])
}
enum LedgerActorType { user agent }
enum LedgerAnchorChain { vibehub zhixin baoquan }

model AigcStamp {
  id                 String   @id @default(cuid())
  artifactId         String   @unique
  mode               AigcStampMode                  // text | image | audio | video
  explicitMark       String                         // 实际加在内容上的提示文字 / 角标描述
  implicitMetadata   Json                           // {provider, contentSerial, generatedAt, ...}
  providerCode       String                         // 服务提供者编码（VibeHub 自有 + 上游 LLM 标识）
  contentSerial      String                         // 内容编号（用于回溯）
  providerApi        AigcProviderApi                // tencent | aliyun | local
  stampedAt          DateTime @default(now())
  retainedUntil      DateTime                       // 标识办法要求 ≥ 6 个月

  @@index([providerApi, stampedAt])
}
enum AigcStampMode { text image audio video }
enum AigcProviderApi { tencent aliyun local }

model OpcProfile {
  id              String   @id @default(cuid())
  userId          String   @unique
  companyKind     OpcCompanyKind                    // solo_dev | design | content | ops | domain | other
  tagline         String?                           // ≤ 120 字
  openToCollab    Boolean  @default(false)
  lookingFor      String[]                          // 标签：[design, frontend, growth, ...]
  publicCardSlug  String   @unique                  // /u/[slug]
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}
enum OpcCompanyKind { solo_dev design content ops domain other }

model OpcTrustMetric {
  userId                 String   @id
  ledgerEntryCount       Int      @default(0)
  snapshotCount          Int      @default(0)
  stampedArtifactCount   Int      @default(0)
  publicWorkCount        Int      @default(0)
  avgResponseHours       Float?
  registrationDays       Int      @default(0)
  lastComputedAt         DateTime @default(now())
}

model LegalAttestationLink {
  id              String   @id @default(cuid())
  ledgerEntryId   String
  chain           LedgerAnchorChain
  txId            String
  proofUrl        String?
  verifiedAt      DateTime
  @@index([ledgerEntryId])
}
```

## 十一. 90 天 PMF 验证

### 11.1 北极星指标（同时满足 = 通过）

1. **合规启用率**：注册用户中"开启 AIGC 自动加标"的比例 ≥ **70%**
2. **月度 Ledger 导出率**：注册 ≥ 30 天的活跃用户中，本月至少导出过 1 次 Ledger 的比例 ≥ **30%**
3. **Pro 转化率**：注册 ≥ 30 天的用户中升级 Pro 比例 ≥ **5%**

### 11.2 辅助指标

- Trust Card 完整度（6 项 Trust 指标 ≥ 4 项有真实数据）≥ **60% 用户**
- Ledger 公开校验调用量（外部用户校验我方 Ledger）≥ **每周 100 次**
- 至信链锚定使用率（Pro 用户中）≥ **40%**

### 11.3 失败处置

- 任意一个北极星 < 50% 目标值 → **不允许**继续加新功能，回到打磨现有 3 件事
- 三个北极星 < 30% 目标值 → 召开"是否解散"评审
